# 02 TDD Workflow

## Cycle

1. Write failing test for new behavior.
2. Implement minimal code to pass.
3. Refactor while tests stay green.

## Scope

1. Domain entity validation
2. Use-case behavior and edge cases
3. Contract-level API checks for critical routes

## Baseline Commands

1. `cd /Users/zqs/Downloads/project/DCF/app && npm test`
2. New code is incomplete if no test covers changed behavior.
