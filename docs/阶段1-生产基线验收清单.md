目标：在进入“单岗位真实上岗”前，先完成最小生产门禁，确保系统可控、可追溯、可回滚。

## 1. 必过门禁

1. 单元/集成测试全通过  
   命令：`cd /Users/zqs/Downloads/project/DCF/app && npm test`
2. `P1` 风险自动回滚演练通过  
   命令：`cd /Users/zqs/Downloads/project/DCF/app && npm run drill:p1-rollback`
3. 审计字段与状态机流转检查通过  
   命令：`cd /Users/zqs/Downloads/project/DCF/app && npm run verify:prod-gates`
4. 生产画像硬闸门通过（鉴权硬化 + 环境硬化 + 可选契约/账本）  
   命令：`cd /Users/zqs/Downloads/project/DCF/app && npm run verify:release`

## 2. 验收标准（建议硬闸门）

1. 连续 7 天无 `P1` 事故
2. 回滚链路演练可复现，且报告可导出
3. 任务事件审计字段完整（`trace_id`、`task_id`、`employee_id`）
4. 任务状态流转无非法跳转

## 3. 运维与应急（上线前）

1. 明确值班责任人（业务 Owner / 技术 Owner / 治理 Owner）
2. 配置告警接收与升级路径
3. 完成“冻结凭证 -> 降级权限 -> 审计排查 -> 恢复”的演练
4. 保证可一键人工接管和停机

## 4. 结果留档

1. 每次门禁检查保存执行时间与执行人
2. 保留最近一周门禁结果快照
3. 对失败项记录根因、修复单号、复验结果
