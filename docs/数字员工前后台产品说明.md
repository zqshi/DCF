# 数字员工前后台产品说明

## 1. 产品分层

1. 前台（Front Stage）
   - 面向业务用户
   - 核心动作：创建数字员工、下发任务、查看任务结果
   - 地址：`/front.html`

2. 后台（Back Office）
   - 面向管理员/运营/治理角色
   - 核心动作：查看员工详情、能力沉淀、子Agent、skills、行为日志
   - 地址：`/admin.html`

## 2. 前台交互模型

1. 用户提交交互任务
2. 服务端父Agent接收并编排执行
3. 执行层（OpenHands适配）完成动作
4. 系统进入自举循环（评估->纠偏->再执行）
5. 前台持续看到状态和结果变化

## 3. 后台管理模型

员工详情包含：

1. 基础信息：`employeeCode`, `name`, `email`, `creator`, `department`, `role`
2. 成长信息：`knowledge`, `capabilities`
3. 结构信息：`childAgents`, `linkedSkillIds`
4. 检索策略：`retrievalPolicy.mode`（`inherit|auto|busy|idle`）
4. 关联资产：`relatedSkills`, `relatedTasks`
5. 行为日志：`logs`（任务创建、运行、纠偏、成功/失败）

## 4. 技能与引入策略

1. Skill 分类
   - `general`: 通用技能
   - `domain`: 领域技能（必须指定领域）

2. 引入策略
   - 系统支持 GitHub 检索候选项目
   - 对候选仓库进行可量化评分
   - 评估后再决定引入为技能或依赖

## 5. API 分区

1. 前台 API
   - `GET/POST /api/front/employees`
   - `GET/POST /api/front/tasks`

2. 后台 API
   - `GET /api/admin/employees`
   - `GET /api/admin/employees/:id`
   - `GET /api/admin/employees/:id/logs`
   - `GET /api/admin/tools/pending`
   - `POST /api/admin/tools/mcp-services/:id/approve`
   - `POST /api/admin/tools/mcp-services/:id/reject`
   - `POST /api/admin/tools/mcp-services/:id/rollback`
   - `POST /api/admin/tools/mcp-services/:id/resubmit`
   - `GET /api/admin/runtime-status`（含检索策略指标 `retrieval.*`）

3. 共享 API
   - `GET/POST /api/skills`
   - `POST /api/skills/link`
   - `GET /api/oss/search`
   - `POST /api/registry/tools/register`（外部工具注册，默认 `pending + disabled`）
   - `POST /api/registry/skills/register`（外部技能注册，默认 `pending`）
   - `GET /api/metrics`（含检索策略计数）

## 6. 迭代约束

1. 前后台职责边界必须稳定
2. 业务用语必须落在领域模型中，不在页面层扩散
3. 新能力必须通过技能分类和质量门禁
4. 自举循环必须可审计、可回放
