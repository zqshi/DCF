# 运行时能力 1:1 对齐清单（屏蔽 OpenClaw 品牌暴露）

## 1. 目标定义

1. 对业务用户与管理员屏蔽 OpenClaw 品牌与内部实现细节。
2. 对外仅暴露 DCF 语义：`Runtime / Execution / Agent / Session`。
3. 通过接口契约、状态机与审计字段实现能力 1:1 对齐，不在 DCF 重写执行引擎。

## 2. 分层边界（DDD-lite）

1. `domain`：员工画像、任务约束、审批规则、成长规则。
2. `application`：任务编排、审批与回滚、事件落账、补偿执行。
3. `infrastructure`：`ExecutionGateway` 适配运行时能力（当前实现映射到 OpenClaw）。
4. `interfaces`：前后台 API 与页面，不直接暴露 OpenClaw 术语。

## 3. 1:1 能力矩阵

1. 任务提交
- Runtime 能力：`submit`
- DCF 对外：`POST /api/front/tasks`
- 对齐字段：`goal/riskLevel/llmConfig/openclaw(agentId/sessionKey/policyId/toolScope/extraSystemPrompt)`
- 验收：任务进入 `pending -> validating` 并生成 `task.created`

2. 状态查询
- Runtime 能力：`status`
- DCF 对外：`GET /api/front/tasks`、`GET /api/admin/tasks/:id`
- 对齐字段：`queued/running/succeeded/failed/aborted`、`runtime.taskId`
- 验收：状态与事件时间线一致，无非法跳转

3. 事件拉取
- Runtime 能力：`events`
- DCF 对外：SSE `/api/events/stream`、轨迹抽屉
- 对齐字段：`runtime.raw.event` + 归一化字段（tool/action/message/details）
- 验收：去重正确、事件顺序稳定、可按 `taskId` 回放

4. 中止执行
- Runtime 能力：`abort`
- DCF 对外：`POST /api/tasks/:id/abort`
- 对齐字段：`task.aborted`
- 验收：中止后不可继续执行

5. Agent 路由与会话隔离
- Runtime 能力：`agentId/sessionKey`
- DCF 对外：员工 `openclawProfile` + 任务覆盖
- 验收：同一员工固定路由，不同员工隔离

6. 提示词治理
- Runtime 能力：`systemPrompt/extraSystemPrompt`
- DCF 对外：员工级配置 + 任务级追加
- 验收：提交体包含合并后的 `extraSystemPrompt`，并保留审计

7. 工具范围与策略
- Runtime 能力：`toolScope/toolPolicy`
- DCF 对外：员工级 `toolScope` + 任务 `llmConfig.toolPolicy`
- 验收：高风险任务命令受审批与策略限制

8. 高风险审批
- Runtime 能力：执行前阻断（由控制层实现）
- DCF 对外：`/api/front/tasks/:id/approve`、`/api/admin/tasks/approve`
- 验收：L4 未达审批阈值不可执行

9. 回滚与补偿
- Runtime 能力：失败后中止（控制层补偿）
- DCF 对外：`rollback/retry-compensation`
- 验收：`P1` 自动回滚、补偿队列与重试/死信完整

10. 成长沉淀
- Runtime 能力：执行输出（控制层沉淀）
- DCF 对外：技能与知识自动沉淀事件
- 验收：`skill.auto.created/link` 与审计链完整

## 4. 页面对齐清单

1. 前台 `front.html`
- 创建员工支持 Runtime Agent/ToolScope/SystemPrompt（可选）
- 对话和轨迹仅显示 Runtime 术语

2. 后台 `admin/*`
- 运行状态页只暴露 `runtimeEnabled/runtimeProvider`
- 员工详情需显示运行时配置与治理配置
- 任务详情需显示运行时快照、审批链、补偿链、审计链

## 5. 契约与测试门禁

1. 契约测试
- `verify-openclaw-contract.js`：运行时握手与版本校验
- 新增要求：对外接口不暴露 OpenClaw 品牌字段

2. 单元/集成测试
- 领域：员工配置继承、任务覆盖、审批规则
- 用例：任务状态机、回滚补偿、事件去重
- 接口：前后台字段稳定性、管理端可观测性

3. 质量门禁
- `npm test`
- `npm run drill:p1-rollback`
- `npm run verify:prod-gates`

## 6. 实施顺序（建议）

1. P0（已开始）
- 引入 `ExecutionGateway` 门面
- 对外接口文案改为 Runtime 中性命名

2. P1
- 后台员工页增加运行时配置编辑与审计
- 后台任务页增加运行时路由/提示词快照显示

3. P2
- 运行时能力差异告警（字段缺失、契约漂移）
- 1:1 对齐仪表板（覆盖率、失败项、回滚率）

## 7. 例外说明

当前不在 DCF 侧重写执行引擎。该策略属于架构决策，不违反既有标准：执行能力通过基础设施适配层接入，控制逻辑仍留在 DCF 领域与应用层。
