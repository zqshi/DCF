# 数字员工实施蓝图（服务依赖与时序）

## 1. 目的

将“数字员工构建方案”转换为可执行工程蓝图，明确：

1. 服务依赖关系
2. 关键业务时序
3. 发布与灰度策略
4. 故障与风险处置流程

## 2. 服务依赖图（文字版）

### 2.1 核心依赖关系

1. `identity-service` -> `policy-service`
2. `orchestrator-service` -> `identity-service`
3. `orchestrator-service` -> `policy-service`
4. `orchestrator-service` -> `execution-service`
5. `execution-service` -> `audit-service`
6. `orchestrator-service` -> `audit-service`
7. `evaluation-service` -> `audit-service`
8. `evaluation-service` -> `skills-registry-service`（P1后）
9. `error-learning-service` -> `evaluation-service`（P1后）

### 2.2 引擎接入关系

1. `orchestrator-service` 通过 `AgentScope Adapter` 接入 AgentScope
2. `execution-service` 通过 `OpenHands Adapter` 接入 OpenHands
3. 平台层通过统一 `Engine Contract` 调用，不直连引擎私有接口

## 3. 关键时序（端到端）

### 3.1 上岗时序

1. 创建数字员工身份
2. 绑定岗位合同
3. 初始化权限与红线规则
4. 发放执行凭证（短期）
5. 开启试运行标记

### 3.2 任务执行时序

1. 业务方发起任务
2. 编排层校验身份与权限
3. 风险判级（是否需要审批）
4. 父Agent分解任务并决定是否创建子Agent
5. 执行层调用 OpenHands 完成动作
6. 回传结果与证据
7. 审计落库与评估打分

### 3.3 错误学习时序

1. 任务失败 -> 错误分级
2. P1直接触发降级/接管
3. P2/P3进入隔离池并生成纠偏任务
4. 纠偏策略在沙箱回放验证
5. 回归通过后灰度发布
6. 观察期通过后转正

## 4. 发布策略

### 4.1 分层发布

1. 编排层与执行层独立发布
2. 治理层规则可热更新
3. 评估层按周期批处理发布

### 4.2 灰度策略

1. 先按任务类型灰度
2. 再按岗位灰度
3. 最后按部门灰度

### 4.3 回滚策略

1. 任意阶段发现 P1 风险立即回滚
2. 回滚优先级：策略回滚 > Skill回滚 > 引擎版本回滚
3. 保留最近三版稳定快照

## 5. 运行策略（自举自动执行）

### 5.1 自动推进规则

1. 满足门槛自动进入下一阶段
2. 不满足门槛自动进入纠偏阶段
3. 连续两周期未改善触发人工复盘

### 5.2 门槛规则（首版）

1. `目标达成率 >= 85%`
2. `P1事故 = 0`
3. `错误复发率环比下降`
4. `Skill复用率环比上升`

## 6. 风险处置手册（首版）

### 6.1 权限类风险

1. 现象：越权调用/异常授权增长
2. 动作：立即冻结凭证 -> 降级权限 -> 审计排查 -> 恢复

### 6.2 执行类风险

1. 现象：失败率激增/耗时异常
2. 动作：触发熔断 -> 切换稳定模板 -> 执行回滚

### 6.3 数据类风险

1. 现象：敏感数据外发或未授权访问
2. 动作：阻断外发 -> 封禁会话 -> 保全审计证据 -> 合规处置

### 6.4 供应链风险

1. 现象：引入依赖高危漏洞或许可冲突
2. 动作：下线依赖 -> 启用替代实现 -> 补丁后再灰度

## 7. 实施排期建议（4个迭代）

### Iteration 1（2周）

1. 完成身份、权限、审计最小闭环
2. 打通单任务执行时序

### Iteration 2（2-4周）

1. 接入 AgentScope + OpenHands 适配层
2. 上线任务编排与回滚机制

### Iteration 3（2-4周）

1. 上线评估服务与错误学习隔离池
2. 上线 Skill灰度与退役流程

### Iteration 4（2-4周）

1. 同部门第二岗位复制
2. 输出第一版复用与ROI报告

## 8. 验收清单

1. 是否可在无人工干预下完成完整任务闭环
2. 是否具备一键接管、熔断、回滚能力
3. 是否能生成周期评估并自动触发纠偏
4. 是否完成至少一个岗位复制案例
5. 是否形成可复用模板（岗位模板、编排模板、执行模板）
