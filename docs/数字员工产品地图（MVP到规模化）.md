# 数字员工产品地图（MVP到规模化）

## 1. 总体形态（1+4+1）

1. 核心平台：`Digital Employee OS`
2. 产品面A：岗位与上岗
3. 产品面B：能力工厂
4. 产品面C：任务编排
5. 产品面D：治理风控
6. 运行内核：`OpenHands + AgentScope`

## 2. 角色地图

1. 业务负责人
   - 定义岗位目标和边界
   - 审批高风险动作
   - 评估业务结果
2. 数字员工运营
   - 维护任务模板
   - 监控执行状态
   - 跟进异常处理
3. 工程负责人
   - 维护原子能力与Skill版本
   - 管理开源引入与回归验证
4. 安全合规负责人
   - 配置权限和红线规则
   - 执行审计与事故复盘
5. 管理层
   - 关注覆盖率、ROI、风险趋势

## 3. 页面清单（按产品面）

### 3.1 岗位与上岗

1. `岗位模板库`
2. `数字员工入职台`
3. `部门归属与禁区配置`
4. `KPI合同配置页`

### 3.2 能力工厂

1. `原子能力库`
2. `Skill工作台`
3. `开源引入评估页`
4. `能力退役页`

### 3.3 任务编排

1. `父Agent编排画布`
2. `子Agent策略页（阈值/TTL/配额）`
3. `任务执行时间线`

### 3.4 治理风控

1. `权限策略中心`
2. `审批中心`
3. `审计追踪页`
4. `熔断与回滚控制台`

### 3.5 评估成长

1. `岗位绩效看板`
2. `错误学习看板`
3. `Skill复用与成本看板`
4. `成长建议页`

## 4. 交互形态

1. 对话式交互
   - 业务负责人与数字员工基于任务上下文对话
   - 返回结果、证据、风险提示
2. 卡片式交互
   - 任务卡、审批卡、异常卡、回滚卡
   - 支持快速操作和状态追踪
3. 画布式交互
   - 父子Agent编排、分支逻辑、失败路径可视化
4. 仪表盘交互
   - 目标达成率、错误复发率、Skill复用率、成本趋势
5. 审批流交互
   - 发起、审核、执行、留痕闭环

## 5. MVP优先级

### 5.1 P0（必须）

1. `岗位模板库`
2. `数字员工入职台`
3. `原子能力库`
4. `任务执行时间线`
5. `权限策略中心`
6. `审计追踪页`
7. `岗位绩效看板（最小3指标）`

### 5.2 P1（MVP后）

1. `Skill工作台`
2. `子Agent策略页`
3. `错误学习看板`
4. `开源引入评估页`

### 5.3 P2（规模化）

1. `能力市场`
2. `跨部门编排中心`
3. `策略自优化模块`
4. `组织级ROI中心`

## 6. 阶段里程碑（与自举路径对齐）

### 6.1 S0 / F0（基线）

交付：

1. 岗位模板库
2. 数字员工入职台
3. 原子能力库
4. 基础审计链路

验收：

1. 单岗位可上岗
2. 可追溯
3. 可停机

### 6.2 S1 / F1（受控执行）

交付：

1. 任务编排画布（最小版）
2. 审批中心（高风险动作）
3. 熔断与回滚机制

验收：

1. 核心任务可稳定执行
2. P1事故为0
3. 错误复发率开始下降

### 6.3 S2 / F2（纠偏沉淀）

交付：

1. Skill工作台
2. 错误学习看板
3. 子Agent策略页

验收：

1. Skill复用率持续上升
2. 回滚率可控
3. 单位成本出现下降趋势

### 6.4 S3 / F3-F4（复制扩张）

交付：

1. 跨部门协作能力
2. 能力市场
3. 组织级运营与ROI看板

验收：

1. 第二岗位上线周期显著缩短
2. 跨部门成功率稳定
3. ROI持续为正

## 7. 基础设施映射（OpenHands + AgentScope）

1. OpenHands（执行引擎）
   - 承担命令、脚本、文件等任务执行
   - 作为Skill运行容器
2. AgentScope（编排引擎）
   - 承担父子Agent任务分解与协作编排
   - 管理会话状态与协同拓扑
3. 平台适配层（Engine Adapter）
   - 屏蔽引擎差异
   - 统一任务协议、状态模型、日志模型

## 8. 关键实施约束

1. 先单岗位闭环，再跨岗位扩张
2. 先治理底线到位，再提升自主度
3. 先模板化沉淀，再规模复制
4. 先业务结果达标，再追求能力丰富度
