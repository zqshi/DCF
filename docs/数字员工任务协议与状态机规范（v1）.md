# 数字员工任务协议与状态机规范（v1）

## 1. 目标

定义编排层（AgentScope）与执行层（OpenHands）之间的统一协议，确保：

1. 可替换
2. 可审计
3. 可回滚

## 2. 任务协议（Task Contract）

### 2.1 输入模型

```json
{
  "task_id": "string",
  "trace_id": "string",
  "employee_id": "string",
  "parent_agent_id": "string",
  "goal": "string",
  "context": {},
  "risk_level": "L1|L2|L3|L4",
  "permission_ticket": "string",
  "constraints": {
    "time_budget_ms": 0,
    "cost_budget": 0,
    "allowed_tools": []
  }
}
```

### 2.2 输出模型

```json
{
  "task_id": "string",
  "trace_id": "string",
  "status": "succeeded|failed|rolled_back",
  "result": {},
  "evidence": [],
  "references": [],
  "cost": {
    "tokens": 0,
    "compute_ms": 0
  },
  "error": {
    "code": "string",
    "message": "string",
    "severity": "P1|P2|P3"
  }
}
```

## 3. 状态机规范

状态流转：

`pending -> validating -> approved(optional) -> running -> succeeded|failed -> rolled_back(optional)`

### 3.1 进入条件

1. `pending -> validating`：任务创建完成
2. `validating -> approved`：高风险任务需要审批
3. `approved -> running`：审批通过且权限票据有效
4. `running -> failed`：执行错误或策略拒绝
5. `failed -> rolled_back`：满足回滚规则
6. `running -> succeeded`：结果与证据完整返回

### 3.2 终止条件

1. 熔断触发
2. 权限票据失效
3. 风险规则命中红线

## 4. 错误码规范

1. `AUTH_*`：认证/凭证错误
2. `POLICY_*`：策略拒绝/审批失败
3. `EXEC_*`：执行失败
4. `RUNTIME_*`：运行时异常
5. `DATA_*`：数据访问异常
6. `COMPLIANCE_*`：合规红线触发

## 5. 审计字段规范

每条事件必须带：

1. `trace_id`
2. `task_id`
3. `employee_id`
4. `parent_agent_id`
5. `child_agent_id`（如有）
6. `event_type`
7. `event_time`
8. `payload_ref`

## 6. 回滚规则

### 6.1 自动回滚触发

1. P1错误
2. 连续失败超过阈值
3. 越权调用命中红线

### 6.2 回滚优先顺序

1. 策略回滚
2. Skill版本回滚
3. 执行模板回滚
4. 引擎版本回滚

## 7. 协议版本管理

1. 协议版本采用 `major.minor.patch`
2. 破坏性变更必须升级 major
3. 编排层与执行层必须支持最近两个 major 版本并行

## 8. 验证清单

1. 协议字段完整性校验
2. 状态机非法跳转校验
3. 审计字段缺失拦截
4. 回滚链路可重放
