# 4周实施计划与验收标准（OpenClaw接入版）

## 0. 关联文档
1. 接口契约：`OpenClaw接入接口契约.md`
2. 迁移清单：`迁移任务清单（可勾选）.md`
3. 现有规范入口：`standards/README.md`

## 1. 目标
在4周内完成“OpenClaw执行层 + DCF企业控制层”可用闭环，避免重建执行引擎，优先交付企业可治理能力。

## 2. 周计划

### Week 1：Runtime接入与最小闭环
#### 交付
1. OpenClaw Adapter接入（任务提交、状态查询、事件拉取、中止）。
2. 任务链路可跑通：前台下发 -> OpenClaw执行 -> DCF记录事件 -> 前台可见状态。
3. 保留本地执行兜底策略（OpenClaw不可用时回退）。

#### 验收标准
1. 成功任务端到端时延`P95 < 10s`（演示环境）。
2. 至少产生`task.created/running/succeeded|failed`事件。
3. 断开OpenClaw后系统仍可执行（降级模式可用）。

### Week 2：身份权限与组织边界
#### 交付
1. 一人一父数字员工约束固化。
2. 任务前置策略校验（部门/岗位/风险级别边界）。
3. 高风险动作（L3/L4）审批占位流程接入。

#### 验收标准
1. 非法岗位任务拒绝率100%。
2. 越权工具调用被拦截并产生日志。
3. 审计可追溯：可按`taskId`回放关键决策链。

### Week 3：技能治理与开源引入
#### 交付
1. Skills自动沉淀流程：创建、关联、分类（general/domain）。
2. GitHub候选检索自动化（由数字员工触发，不由人手工挂载）。
3. Skill提案审批流（pending/approved/rejected/rollback）。

#### 验收标准
1. 每个skill都有来源、版本、风险评分、状态。
2. 引入失败可回滚且不影响主任务执行。
3. 技能复用率指标可观测（至少可按周统计）。

### Week 4：运营可观测与上线准备
#### 交付
1. 前台：会话线程、任务阶段、轨迹抽屉、会话复盘。
2. 后台：员工详情、技能台账、任务台账、行为日志、运行状态。
3. 基线压测与上线手册。

#### 验收标准
1. 核心接口错误率`< 1%`（测试压测场景）。
2. 审计事件完整率`100%`（抽样）。
3. 演示场景可稳定运行`24h`无阻断故障。

## 3. 质量门禁（每周必过）
1. 单测通过率100%，新增逻辑必须先补测试（TDD）。
2. 接口契约测试通过，禁止前后端隐式字段漂移。
3. Lint/格式检查通过。
4. 关键文档同步更新：接口、流程、异常处理。

## 4. 指标体系（上线前）
1. 任务成功率
2. 纠偏触发率
3. 复发错误率
4. 技能复用率
5. 平均交付时长
6. P1事件数
7. 审计覆盖率

## 5. 上线闸门（Go/No-Go）
1. 安全：权限绕过与未审计路径为0。
2. 稳定：连续24小时回归与巡检通过。
3. 可回退：OpenClaw故障可降级到本地执行。
4. 可运维：日志、指标、告警、排障手册齐全。
